Performance Testing

TypeWhat It Tests
LoadUnder expected/normal load conditions
StressUnder extreme load — beyond normal limits
SpikeUnder sudden, dramatic increases in load
EnduranceOver an extended period of time
ScalabilityAbility to scale up or down based on demand
VolumeManaging and processing large volumes of data
ConcurrencyMultiple users or processes accessing simultaneously
SoakIdentify potential memory leaks and performance degradation over time
ConfigurationPerformance under various hardware/software configuration settings
IsolationIndependently identify effects of various system components
Tools & Examples

Tools: Apache JMeter / LoadRunner (load tests) · AWS/Azure (scaling simulation) · Selenium Grid (concurrent users)

Example tests: Simulate 1,000 users browsing and purchasing during a Black Friday sale. Run a social media application at average user load for a week to detect memory leaks.

Usability Testing

TypeFocus
User InterfaceVisual elements and layout of the application
NavigationEase of navigation and clarity of instructions
AccessibilityAccessible to users with disabilities
User Experience (UX)Gather user feedback to understand overall experience
Error HandlingHow software handles user errors and provides feedback
ConsistencyDesign and behaviour are consistent throughout
Help & DocumentationQuality and usefulness of help files and documentation
ResponseHow quickly the software responds to user actions
MultilingualUsability in different languages
DeviceEase of use on various devices (smartphones, tablets)
Tools & Examples

Tools: Google Lighthouse (accessibility) · Maze (prototype validation) · UserTesting (remote usability, video feedback, qualitative insights)

Example tests: Test form validation and error messages in a registration form. Test a multilingual website for accurate translations and proper text alignment.

Security Testing

TypeFocus
PenetrationSimulating attacks to identify security vulnerabilities
VulnerabilityAutomated tools to detect known vulnerabilities quickly
Access ControlUser access levels are correctly implemented and enforced
AuthenticationVerifies authentication mechanisms — passwords, MFA
EncryptionValidates implementation of encryption methods
Session ManagementHow the software manages user sessions
Intrusion DetectionEffectiveness of IDS and monitoring tools
DoS TestingSimulates Denial-of-Service attacks
Data IntegrityEnsures data remains accurate, consistent, and unaltered
Network SecuritySecurity of the network infrastructure
Tools & Examples

Tools: Nessus / Qualys / OpenVAS (vulnerability scans) · OWASP ZAP (XSS, SQL injection) · Metasploit (penetration testing)

Example tests: Simulate a cyber-attack to test IDS response. Verify integrity of financial transaction data in a banking application.

Risk Assessment

Three Pillars
  • Security Auditing — Reviews code and configurations to ensure adherence to best practices.
  • Compliance — Adherence to industry standards and regulations (GDPR, POPIA, HIPAA); avoiding penalties and legal consequences.
  • Security Configuration — Security settings and configurations are correctly applied across all systems.
Tools & Examples

Tools: RiskLens (cyber risk quantification using FAIR model) · Rapid7 InsightVM (vulnerability management with risk prioritisation; integrates into CI/CD)

Example tests: Audit cloud-based application security configurations. Conduct a security audit of a banking application against security standards. Test a healthcare app for HIPAA compliance. Check for GDPR/POPIA data jurisdiction compliance.

Reliability Testing

TypeFocus
RecoveryAbility to recover from failures and crashes
FailoverPerformance during hardware or software failures
Reliability GrowthRepeated testing cycles measuring improvements in reliability over time
Continuous OperationRuns software continuously over an extended period
MTBFMean Time Between Failures — average time between system failures
MTTRMean Time To Repair — average time to repair and restore after a failure
RedundancyEffectiveness of redundant components in maintaining system operation
Error Detection & CorrectionAbility to detect and automatically correct errors
Tools & Examples

Tools: Chaos Monkey (tests resilience by killing random services/instances) · Nagios / Zabbix / Prometheus (uptime tracking and monitoring)

Example tests: Test a video conferencing app on various network conditions (3G, 4G, 5G, Wi-Fi). Calculate MTBF for a data centre's server infrastructure to schedule proactive maintenance.

Maintainability Testing

TypeFocus
Code QualityQuality and readability of the code
DocumentationSoftware documentation is clear and comprehensive
ModularityDesign is divided into well-defined, independent modules
Change ImpactPotential impacts of changes — which parts will be affected by updates
Relevant Architecture Styles

Layered (n-Tier) · Client-Server · Service-Oriented · Event-Driven · MVC · MVVM · Microservices · Monolithic · MVT

Tools & Examples

Tools: SonarQube / ESLint (static code analysis) · Sphinx / Doxygen (documentation generation) · JIRA / TFS (tracking changes and dependencies)

Example tests: Refactor a monolithic application into microservices to improve maintainability. Test automation of routine operations.

Portability Testing

TypeFocus
InstallationEase of installing and setting up the software
AdaptabilityFunctions effectively under various hardware/software configurations
UninstallationCan be cleanly uninstalled without residual files or side effects
MigrationAbility to transfer from one environment to another (e.g. local → cloud)
Backward/Forward CompatibilityFunctions correctly with future and past versions
Driver CompatibilityCompatible with various device drivers
Tools & Examples

Tools: Selenium + BrowserStack (cross-browser tests) · Appium (mobile portability) · Kubernetes + Helm (deployment portability)

Example tests: Deploy a web application on AWS, Azure, and Google Cloud to test adaptability. Test migration of a database from on-premises to a cloud-based service.

Complete Non-Functional Test Type List
