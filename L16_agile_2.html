UP Airlines — Background

UP Airlines is a new player in air travel, currently selling tickets through a global network of travel agents. In the new financial year, the company is building an online ticket search and booking system.

Key Requirements
  • Allow customers to directly book tickets in real-time.
  • Migrate the existing travel agent network to the new platform.
  • Compete with travelstart.co.za, kayak.com, expedia.com.

Scope

Departure/destination airports · One-way and round trips · Class (business, economy, first) · Dates · Seat preference (aisle, middle, window) · Seat location (front, wing, tail) · Number of seats (1–6) · Emergency exit seating restrictions · Price comparison · Hotels, accommodation, rental cars · Real-time reporting and price adjustments.

Exercise 2 — Epics

Each PBI is broken into smaller, deliverable user stories (epics):

Search for a Flight

  • Search one-way, return, and multi-city trips
  • Find cheapest flight in ±3-day range; monthly cheapest; current specials
  • Search by specific airline, by class (Economy/Business/Premium), by number of tickets (1–6)

Compare Flights

  • List all available flights; filter cheapest to most expensive
  • Bookmark a flight; change flight parameters; view flight details
  • View shorter lay-overs; order by price or preferred airline
  • View cost breakdown: base cost, taxes, fuel levy, baggage weight allowance

Complete Booking

  • Add personal information (name, DOB, ID/passport)
  • Add contact details; register an account
  • Send OTP/verification to mobile/email; login; add rewards account

Add Extras

  • Seat allocation (aisle/middle/window) and seat location (front/wing/tail)
  • Extra baggage, hotel accommodation, car rental

Pay for the Ticket

  • Credit card, EFT, loyalty/frequent flyer points, voucher

Receive Notifications

  • Confirm booking, receive booking confirmation, receive invoice

Change Booking

  • Search by booking reference, flight number, name, or ID/passport
  • Change dates (fees may apply), seat allocation, food preference
  • Add car rental or accommodation

Exercise 3 — User Stories & Acceptance Criteria

User StoryValue / SizeAcceptance Criteria
As a traveller, I want to search for flights based on departure, destination, dates, adult/child, and number of travellers, so that I can find suitable flight options. 40 / Large
  • Only five basic inputs required
  • Validate that dates are in the future
  • Validate that departure and destination are not the same
As a traveller, I want to see flights based on my search criteria, so that I can find suitable options. 20 / Medium
  • Show departure, destination, date, time, total time, price
  • Number of connections is out of scope for this story
  • Check that dates correspond to search
As a traveller, I want flights sorted by price by default, so that I can choose the most affordable option. 5 / Small
  • Flights displayed from lowest to highest price
As a traveller, I want to choose between economy, business, and first class, so that I can travel more comfortably. 5 / Small
  • Three options: Economy, Business, First
  • Display as a dropdown on the search page

Exercise 4 — Definition of Done

Writing Test Cases

Golden Rule

For every acceptance criterion, write at least one positive (happy path) AND one negative (failure/edge case) test case.

Example: Pay with a Credit Card

Story: Pay with a credit card  |  AC: Accept Visa & Mastercard only · Validate expiry, CVV · Decline Amex/Diners

TypeInput / ScenarioExpected Result
✅ PositiveValid Visa — correct number, expiry, CVVPayment Successful page shown
✅ PositiveValid Mastercard — correct detailsPayment Successful page shown
✅ PositiveValid card + OTP prompt → accept OTPPayment confirmed
❌ NegativeExpired cardError: 'Card has expired, please use a valid card'
❌ NegativeInvalid CVVError message displayed
❌ NegativeCard number <16 or >16 digitsValidation error shown
❌ NegativeInsufficient fundsError: 'Insufficient funds'
❌ NegativeAmex card (starts 3711…)Error: 'Card type not accepted'
❌ NegativeDiners Club cardError: 'Card type not accepted'
Using AI to Generate Test Cases

Example prompt: "Create test cases for: As a traveller, I want to pay with a credit card. AC: Accept Visa & Mastercard only, validate expiry & CVV, reject Amex."

AI output is a starting point, not a finished test suite. Always review and add domain knowledge and edge cases it misses.

Quick Reference — Agile Concepts

Core Concepts at a Glance
ConceptKey Point
User StoryAs a [user], I want [feature], so that [value]
Acceptance CriteriaGiven / When / Then — conditions that must pass for a story to be done
Definition of DoneAgreed, realistic, 5–8 items, visible, updated regularly
INVESTIndependent · Negotiable · Valuable · Estimatable · Small · Testable
Test case typesPositive (happy path) + Negative (failure/edge cases) for every AC
EpicLarge PBI broken into multiple smaller user stories
SprintFixed time-box (usually 2 weeks) for delivering working software
VelocityStory points completed per sprint — the team's measurable pace
Story PointsRelative effort — not hours. Estimated collaboratively by the team.
